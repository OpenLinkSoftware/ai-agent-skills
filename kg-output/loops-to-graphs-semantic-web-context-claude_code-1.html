From Loops to Graphs — and Why the Graph Was Always a Web

Thesis & Evidence

Testing the Thesis

The migration from loop engineering to graph engineering independently rediscovers the vision of purpose-specific Semantic Webs: entity relationship graphs become webs when hyperlinks name both entities and relationships, and relationship semantics are given context by ontologies constructed in the same webby form.
— Kingsley Uyi Idehen

The thesis: graphs become webs when names become hyperlinks

The migration from loop engineering to graph engineering is real, but it stops one step short. A graph whose nodes and edges are named by private convention is still a silo; a graph whose entities and relationships are named with hyperlinks — and whose relationship semantics are informed by ontologies built in the same webby form — is a web. That is what a Semantic Web is, and the agent-architecture discourse is rediscovering its premises one failure mode at a time.

Evidence 1 — Perez: four failures of the loop, one failure of the graph

Perez shows the single loop failing four ways — Goodhart gaming, reference blindness, loop conflict, measurement decay — and shows the graph of loops answering each topologically: pairing, hierarchy, arbitration, audit. Then he turns the same knife on the graph: built without anchors, it fails circularly, an elaborate network of mutual confirmation where everything is consistent and nothing is verified. His conclusion: the durable axis is ungrounded versus grounded.

Evidence 2 — agent-rdf-memory: a working semantic web of loops

The agent-rdf-memory repository is an existence proof of the grounded alternative: an AI agent's behavioral contract as 105 hyperlink-named steps across 9 themes, hub-and-spoke from preferences.ttl to 40+ companion specifications, with identity in core.ttl, ontology-routed SPARQL context selection, and episodic session memory. Every rule is an entity with an IRI; every cross-reference is a resolvable link; the audit loop is a SPARQL query. The improvement graph is not merely drawn — it is dereferenceable.

Evidence 3 — Kingsley: identity and preferences are the original context

Unix handled identity and preferences as fundamental components of the bridge between users and computer services — a principle blurred by decades of command-line and graphical interfaces. Langulators (large language models) now offer a multimodal, multilingual interface for driving computers, and the fundamentals of session context via profiles risk being lost in AI's repositioning noise. Recapturing identity and preferences as the foundation for context — an operating framework for LLMs, agents, skills, and data spaces — is one of AI's most powerful benefits, and RDF is the naming discipline that makes that context portable, queryable, and web-grounded. This is exemplified concretely by agent-rdf-memory: its preferences.ttl includes the identity and preference HowTos that are re-established at the start of every new session — the Unix profile principle, made machine-computable.

Beyond LLM Wikis: from Markdown prose to machine-computable sentences

Andrej Karpathy's LLM Wiki idea — feed in documents, get back a structured, explorable wiki — was the spark. A Semantic Web is the next step: don't stop at Markdown prose. Any document identified by a URL becomes a living knowledge graph — entities and relationships named with hyperlinks, grounded in Linked Data principles, queryable via SPARQL, navigable follow-your-nose style, and continuously enriched by AI agents on autopilot. The missing piece was always the client — and just as Mosaic and Netscape were the HTML clients that made the document Web practical, LLMs are the generic RDF clients that make purpose-specific Semantic Webs practical.

Verdict: confirmed — human judgment, codified machine-computably

The thesis holds. The loops-to-graphs shift is a Semantic Web's premises arriving through the back door of agent engineering: graphs need names, names need meaning, meaning needs ontologies, and ontologies must themselves be webby or the grounding terminates in a private convention. The root judgment of what "better" means originates with people — and workflows informed by a Semantic Web codify that human judgment in machine-computable form. This is the LLM Wiki pattern taken beyond Markdown prose: knowledge bases (or graphs) encoded by AI agents using RDF notations and Linked Data principles, with judgments signed, attributed, and dereferenceable to their authors via verifiable identity.

Guide

How to Ground an Agent's Improvement Graph in a Semantic Web

  1. Name entities and relationships with hyperlink IRIs

    Give every node and every edge in the improvement graph a dereferenceable HTTP IRI, so any claim, metric, or rule can be followed to its definition and source.

  2. Encode identity and preferences as RDF profiles

    Recapture the Unix fundamentals: represent who the agent works for and how it must behave as queryable RDF — identity plus a preferences-based behavioral contract — not as flat prose in a vendor silo.

  3. Give relationship semantics an ontology

    Describe what the graph's edges mean in a lightweight, webby ontology — published, hyperlinked, and inspectable — so conflicting loops share a vocabulary and trade-offs become expressible.

  4. Route context by intent through the ontology

    Classify each prompt's intent and select session context via ontology-routed SPARQL queries over named graphs, expanding to detailed specifications only when the intent requires them.

  5. Pair every optimizing metric with an independent watcher

    Store counter-metrics and audit data in separately named, separately governed graphs the optimizing loop cannot rewrite, and make the audit loop a standing SPARQL ASK query.

  6. Anchor in verifiable identity and provenance

    Sign and attribute judgments with verifiable identity (WebID) and record derivation with provenance vocabulary (PROV-O), so grounding resolves to accountable authors and inspectable sources rather than private convention.

  7. Keep the root judgment human — and record it

    The definition of "better" is chosen, not computed. Record that choice as an attributed, dereferenceable claim so the improvement machinery is honest about where its own authority ends.

FAQ

Frequently Asked Questions

What did Peter Steinberger's nine-word post crystallize?

It crystallized a field-wide transition in AI agent architecture: builders recognized themselves mid-stride between the single self-improvement loop pattern they were leaving and the graph-of-loops pattern they were reaching for.

What are the four failure modes of the single improvement loop?

Goodhart gaming (the loop optimizes a metric that detaches from its meaning), reference blindness (nothing in the loop can question its own target), loop conflict (independently built loops fight each other), and measurement decay (the loop's own sensors and definitions rot while dashboards stay green).

What is a graph of loops?

A network of improvement cycles that watch, feed, constrain, and correct one another — paired counter-metric loops, hierarchical reference-owning loops, arbitration loops, and audit loops. The reliability lives in the edges: which loop feeds, watches, or vetoes which.

Why do graphs of loops still fail?

They fail circularly: when every loop watches another loop and no loop touches the ground, the graph becomes a network of mutual confirmation in which everything is consistent and nothing is verified. Topology buys sophistication, not contact with reality — the graph needs anchors.

How does a Semantic Web answer the anchor problem?

By naming entities and relationships with dereferenceable hyperlink IRIs: a claim can be followed to its source, identity is verifiable (WebID), provenance is explicit (PROV-O), and the ontologies that give relationships meaning are themselves webby — hyperlinked, resolvable, inspectable — so grounding does not terminate in a private convention.

How does agent-rdf-memory demonstrate the thesis?

It is a working semantic web of loops: an AI agent's behavioral contract encoded as 105 hyperlink-named steps across 9 themes in RDF-Turtle, hub-and-spoke from preferences.ttl to 40+ companion specifications, with identity in core.ttl, ontology-routed SPARQL context selection, and episodic session memory. Every rule is a dereferenceable entity, and the audit loop is a SPARQL query.

What role do identity and preferences play in agent context?

They are the original context mechanism: Unix treated identity and preferences as fundamental components of the bridge between users and computer services. Session context via profiles is that same principle, and encoding profiles in RDF makes agent context portable, queryable, and grounded in the web rather than trapped in a vendor silo.

What is a langulator?

Kingsley Idehen's term for a large language model viewed as a language-driven calculator: a multimodal, multilingual natural-language interface for driving computers, succeeding the command-line and graphical interfaces that previously blurred the identity-and-preferences fundamentals of computing.

What is an LLM Wiki, and how does a Semantic Web go beyond it?

An LLM Wiki, an idea popularized by Andrej Karpathy, feeds documents to a language model and gets back a structured, explorable wiki — typically Markdown prose. A Semantic Web goes one step further: the same documents become living knowledge graphs of explicit machine-computable sentences, with entities and relationships named by hyperlinks, grounded in Linked Data principles, queryable via SPARQL, and continuously enriched by AI agents.

Why are LLMs described as generic RDF clients?

RDF adoption long lagged for want of a usable generic client — the "I'll know it when I see it" paradox. Just as Mosaic and Mozilla/Netscape were the HTML clients that made the document Web practical, LLMs fill that role for RDF: translating structured data into natural-language insights, meshing disparate sources, anchoring outputs to reliable data to mitigate hallucination, and traversing triples follow-your-nose style.

Glossary

Key Terms

Semantic Web
An extension of the World Wide Web in which information is given well-defined meaning via RDF: entity relationship graphs become webs because hyperlinks name entities and relationships, and relationship semantics are informed by ontologies constructed in the same webby form. There is no single definite-article Semantic Web — there are purpose-specific Semantic Webs, each a deployment instance ("a Semantic Web").
Linked Data
The publishing discipline that manifests a Semantic Web: name things with HTTP IRIs, make the names dereferenceable, and return useful, further-linked descriptions when they are looked up.
Ontology
A formal model of classes, properties, and relationships shared by a community of interest — the schema layer that gives relationship semantics their context, itself published in webby, hyperlinked form.
RDF (Resource Description Framework)
The W3C framework for describing entities and relationships as subject-predicate-object statements whose components are hyperlink IRIs — the naming substrate that turns graphs into webs.
SPARQL
The W3C query language for RDF graphs; in an improvement graph, the audit loop is a SPARQL query over independently named and governed graphs.
Loop Engineering
The craft of building one clean self-improvement cycle: choose a measurable thing, set a reference, close the cycle, resist fiddling between measurements. The previous era's core skill in agent building.
Graph Engineering
The emerging craft of loop architecture: pairing metrics with watchers, giving references owners, separating loop speeds, and making some loop answer for reality — designing the network of cycles rather than the cycle.
Anchor
A measurement or rule the improvement machinery cannot argue with or tune: revenue that landed in the bank, tests that actually executed, frozen rules, held-out sets. In a Semantic Web's terms: names that dereference to verifiable, provenance-bearing ground truth.
Langulator
A large language model viewed as a language-driven calculator: a multimodal, multilingual natural-language interface for driving computers. Coined by Kingsley Uyi Idehen.
Behavioral Contract
An agent's standing instructions encoded as queryable, hyperlink-named RDF entities rather than flat prose — rules that are resolvable, auditable via SPARQL, and harder to overlook than markdown.
Profile (Session Context)
The pairing of identity and preferences that constitutes session context — a fundamental component of the user-to-services bridge since Unix, now the foundation for LLM and agent context.
Meshup
Semantic content fusion: combining and enriching information from multiple sources using Linked Data and RDF to produce a semantically coherent, queryable knowledge graph — as this document does with five sources.
LLM Wiki
An emerging pattern, popularized by Andrej Karpathy, in which documents are fed to a language model that returns a structured, explorable wiki. A Semantic Web takes it beyond Markdown prose: the wiki's content becomes explicit machine-computable sentences — knowledge bases (or graphs) encoded by AI agents using RDF notations and Linked Data principles.
Generic RDF Client
The long-missing piece that kept RDF adoption lagging. LLMs now fill the role — as Mosaic and Mozilla/Netscape did for HTML — by translating structured data into natural language, meshing disparate sources, anchoring outputs to verifiable relationships, and navigating triples follow-your-nose style.
